# 训练任务 API (Training Runs)

> **路由前缀：** `/api/v2/training-runs`  
> **标签：** `training-runs`

训练任务（Training Run）是平台的核心资源，管理从创建、排队、执行到完成的全生命周期。

---

## 1. 获取训练任务列表

```
GET /api/v2/training-runs
```

### 请求参数 (Query)

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码 |
| `page_size` | int | 否 | 50 | 每页条数 (1~500) |
| `project_id` | int | 否 | - | 按项目过滤 |
| `dataset_id` | int | 否 | - | 按数据集过滤 |
| `architecture_id` | int | 否 | - | 按网络架构过滤 |
| `status` | string | 否 | - | 按状态过滤：`created`/`queued`/`running`/`completed`/`failed`/`cancelled`/`deleted` |
| `include_hidden` | bool | 否 | false | 是否包含已隐藏的任务 |

### 响应

分页返回 `TrainingRunOut` 列表。

---

## 2. 创建训练任务

```
POST /api/v2/training-runs
```

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_id` | int | ✅ | 项目 ID |
| `architecture_id` | int | ✅ | 网络架构 ID |
| `dataset_version_id` | int | 否 | 数据集版本 ID（不传则使用数据集的活跃版本） |
| `name` | string | 否 | 任务名称（最长 255 字符） |
| `parameters` | object | ✅ | 训练参数（见下表） |

#### parameters 参数

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `epochs` | int | 100 | 训练轮数 (>0) |
| `batch_size` | int | 16 | 批大小；设 -1 可开启 Ultralytics 自动批大小 |
| `image_size` | int | 640 | 输入图片尺寸 (>0) |
| `learning_rate` | float | 0.01 | 学习率 (>0) |
| `patience` | int | 50 | 早停耐心值 (≥0) |
| `device` | string | `auto` | 设备：`auto` / `cpu` / `gpu` / `cuda:0` / `0,1`（多 GPU） |
| `workers` | int | 8 | 数据加载线程数 (≥-1) |
| `use_pretrained` | bool | true | 是否使用预训练权重 |
| `optimizer` | string | `AdamW` | 优化器 |
| `augmentation` | object | null | 数据增强参数 |
| `additional_params` | object | null | 额外框架参数 |

### 请求示例

```json
{
  "project_id": 1,
  "architecture_id": 2,
  "name": "yolov8n-exp01",
  "parameters": {
    "epochs": 100,
    "batch_size": 16,
    "image_size": 640,
    "learning_rate": 0.01,
    "patience": 50,
    "device": "0",
    "optimizer": "AdamW"
  }
}
```

### 响应

- **状态码：** `201 Created`
- **响应体：** `TrainingRunOut` 对象

## 5. 任务生命周期

### 5.1 加入队列

```
POST /api/v2/training-runs/{run_id}/queue
```

将 `created` 状态的任务放入训练队列。

### 5.2 恢复训练

```
POST /api/v2/training-runs/{run_id}/resume
```

恢复已暂停或失败的任务。

### 5.3 取消训练

```
POST /api/v2/training-runs/{run_id}/cancel
```

**请求体（可选）：**

```json
"取消原因描述"
```

### 5.4 删除训练任务

```
DELETE /api/v2/training-runs/{run_id}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `force` | bool | false | 级联删除关联模型版本和部署 |

## 7. 训练指标

### 7.1 获取 Epoch 指标

```
GET /api/v2/training-runs/{run_id}/metrics/epochs
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `limit` | int | 5000 | 返回条数 (1~100000) |
| `source` | string | `auto` | 数据源：`auto` / `db` / `mlflow` |

**响应：**

```json
[
  {
    "metric_id": 1,
    "run_id": "abc-123",
    "epoch": 1,
    "metrics": {
      "train/box_loss": 0.05,
      "train/cls_loss": 0.03,
      "metrics/mAP50": 0.65,
      "metrics/mAP50-95": 0.42
    },
    "created_at": "2026-03-01T10:05:00+00:00"
  }
]
```

### 7.2 指标实时流 (WebSocket)

```
ws://<host>/api/v2/training-runs/{run_id}/metrics/stream?from_metric_id=0&from_event_id=0
```

#### 消息类型

| type | 说明 |
|------|------|
| `cursor` | 初始游标信息 |
| `status` | 状态/进度变更（status, progress, current_epoch） |
| `metric` | 新增 epoch 指标 |
| `event` | 新增事件 |
| `done` | 任务已结束 |
| `ping` | 心跳包（每 15 秒） |
| `error` | 错误消息 |

#### status 消息示例

```json
{
  "type": "status",
  "data": {
    "run_id": "abc-123",
    "status": "running",
    "progress": 45,
    "current_epoch": 45,
    "total_epochs": 100,
    "worker_id": "worker-yolo-01"
  }
}
```

#### metric 消息示例

```json
{
  "type": "metric",
  "data": {
    "metric_id": 45,
    "epoch": 45,
    "metrics": { "metrics/mAP50": 0.78 },
    "progress": 45
  }
}
```

## 8. 训练日志

### 8.1 获取日志尾部

```
GET /api/v2/training-runs/{run_id}/logs/tail
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `which` | string | `stdout` | 日志类型：`stdout` / `stderr` |
| `lines` | int | 200 | 返回行数 (1~20000) |

**响应：**

```json
{
  "run_id": "abc-123",
  "which": "stdout",
  "lines": 200,
  "text": "Epoch 45/100...\n..."
}
```

### 8.2 日志实时流 (WebSocket)

```
ws://<host>/api/v2/training-runs/{run_id}/logs/stream?which=stdout&tail=200
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `which` | string | `stdout` / `stderr` / `both` |
| `tail` | int | 初始回溯行数 (0~5000，默认 200) |

#### 消息类型

| type | 说明 |
|------|------|
| `log` | 日志行；`data.mode` 为 `tail`（历史）或 `append`（实时） |
| `done` | 任务结束 |
| `error` | 错误 |

## 10. 模型导出

```
POST /api/v2/training-runs/{run_id}/export
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `format` | string | 否 | 格式：`pt`（原始权重）/ `onnx` |
| `weights` | string | 否 | 权重选择：`best`（默认）/ `last` |
| `opset` | int | 否 | ONNX opset 版本 (9~20) |
| `dynamic` | bool | 否 | ONNX 动态轴（默认 true） |
| `imgsz` | int | 否 | ONNX 图片尺寸 (32~4096) |

**响应：**

```json
{
  "run_id": "abc-123",
  "format": "onnx",
  "weights": "best",
  "download_url": "/static/training/abc-123/weights/best.onnx",
  "artifact": {
    "artifact_id": 2,
    "run_id": "abc-123",
    "kind": "export",
    "name": "best.onnx",
    "path": "abc-123/weights/best.onnx",
    "size_bytes": 12345678,
    "created_at": "2026-03-01T12:30:00+00:00"
  }
}
```

## 11. 训练对比

```
POST /api/v2/training-runs/compare
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `run_ids` | string[] | ✅ | 训练任务 ID 列表（2~20 个） |

**响应：**

```json
{
  "runs": [
    {
      "run_id": "abc-123",
      "name": "yolov8n-exp01",
      "status": "completed",
      "project_id": 1,
      "dataset_version_id": 3,
      "architecture_id": 2,
      "created_at": "2026-03-01T10:00:00+00:00",
      "engine": "ultralytics",
      "framework_key": "pytorch",
      "parameters": { "epochs": 100, "batch_size": 16 },
      "best_metrics": { "mAP50": 0.85 },
      "metric_summary": {
        "best": { "mAP50": 0.85 },
        "final": { "mAP50": 0.83 },
        "source": "db"
      },
      "model_size_mb": 12.5,
      "inference_time_ms": 8.5
    }
  ],
  "parameter_diff": {
    "batch_size": { "abc-123": 16, "def-456": 32 }
  }
}
```

> **注意：** 不同框架（如 PyTorch 与 PaddlePaddle）的任务对比会返回 `409 Conflict`，并附带框架分组信息。

## 数据模型

### TrainingRunOut

| 字段 | 类型 | 说明 |
|------|------|------|
| `run_id` | string | 任务 ID (UUID) |
| `project_id` | int | 项目 ID |
| `dataset_version_id` | int | 数据集版本 ID |
| `architecture_id` | int | 架构 ID |
| `name` | string | 名称 |
| `status` | string | 状态 |
| `progress` | int | 进度百分比 (0~100) |
| `current_epoch` | int | 当前 epoch |
| `total_epochs` | int? | 总 epoch 数 |
| `queued_at` | datetime? | 入队时间 |
| `claimed_at` | datetime? | 被 worker 领取时间 |
| `worker_id` | string? | 执行 worker ID |
| `pid` | int? | 训练进程 PID |
| `heartbeat_at` | datetime? | 最后心跳 |
| `cancel_requested_at` | datetime? | 取消请求时间 |
| `cancel_reason` | string? | 取消原因 |
| `hidden` | bool | 是否已隐藏 |
| `run_dir` | string? | 训练输出目录 |
| `started_at` | datetime? | 开始时间 |
| `finished_at` | datetime? | 结束时间 |
| `error_message` | string? | 错误信息 |
| `parameters` | object? | 训练参数 |
| `result` | object? | 训练结果（权重路径、最终指标等） |
| `meta` | object? | 元信息 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |
