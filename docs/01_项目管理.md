# 项目管理 API (Projects)

> **路由前缀：** `/api/v2/projects`  
> **标签：** `projects`

项目（Project）是训练任务的组织单元，每个项目绑定一个数据集，并指定任务类型。

---

## 1. 获取项目列表

```
GET /api/v2/projects
```

### 请求参数 (Query)

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码 |
| `page_size` | int | 否 | 50 | 每页条数 (1~500) |
| `dataset_id` | int | 否 | - | 按数据集 ID 过滤 |

### 响应示例

```json
{
  "items": [
    {
      "project_id": 1,
      "name": "行人检测项目",
      "description": "城市场景行人目标检测",
      "dataset_id": 5,
      "task_type": "detection",
      "created_by": "admin",
      "tags": {"priority": "high"},
      "is_active": true,
      "created_at": "2026-03-01T10:00:00+00:00",
      "updated_at": "2026-03-01T10:00:00+00:00"
    }
  ],
  "meta": {
    "page": 1,
    "page_size": 50,
    "total": 1
  }
}
```

---

## 2. 创建项目

```
POST /api/v2/projects
```

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 项目名称 (1~255 字符) |
| `dataset_id` | int | ✅ | 关联数据集 ID |
| `task_type` | string | ✅ | 任务类型：`detection` / `segmentation` / `classification` |
| `description` | string | 否 | 项目说明 |
| `created_by` | string | 否 | 创建者 |
| `tags` | object | 否 | 自定义标签 |

### 请求示例

```json
{
  "name": "行人检测项目",
  "dataset_id": 5,
  "task_type": "detection",
  "description": "城市场景行人目标检测"
}
```

### 响应

- **状态码：** `201 Created`
- **响应体：** `ProjectOut` 对象

---

## 3. 获取项目详情

```
GET /api/v2/projects/{project_id}
```

### 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `project_id` | int | 项目 ID |

### 响应

- **状态码：** `200 OK`
- **响应体：** `ProjectOut` 对象

---

## 4. 更新项目

```
PATCH /api/v2/projects/{project_id}
```

### 请求体（仅传需要修改的字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 项目名称 |
| `description` | string | 项目说明 |
| `tags` | object | 自定义标签 |
| `is_active` | bool | 是否启用 |

### 响应

- **状态码：** `200 OK`
- **响应体：** 更新后的 `ProjectOut` 对象

---

## 5. 删除项目

```
DELETE /api/v2/projects/{project_id}
```

### 请求参数 (Query)

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `force` | bool | false | 是否级联删除关联的训练任务和模型版本 |

### 响应

```json
{
  "ok": true,
  "message": "Project deleted"
}
```

## 6. 获取项目模型大小统计

### 6.1 单个项目

```
GET /api/v2/projects/{project_id}/model-size
```

**响应：**

```json
{
  "project_id": 1,
  "completed_models_count": 5,
  "total_size_mb": 125.67
}
```

### 6.2 批量项目

```
GET /api/v2/projects/model-sizes
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_ids` | string | 否 | 逗号分隔的项目 ID 列表，不传则返回所有项目 |

**请求示例：**
```
GET /api/v2/projects/model-sizes?project_ids=1,2,3
```

**响应示例：**

```json
[
  {"project_id": 1, "completed_models_count": 5, "total_size_mb": 125.67},
  {"project_id": 2, "completed_models_count": 3, "total_size_mb": 88.10},
  {"project_id": 3, "completed_models_count": 0, "total_size_mb": 0.0}
]
```

## 7. 对比基线管理

对比基线（Compare Baseline）用于指定每个框架下的基准训练任务，供前端进行多次实验的对比参考。

### 7.1 获取对比基线

```
GET /api/v2/projects/{project_id}/compare-baseline
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `framework_key` | string | ✅ | 框架标识：`pytorch` / `paddle` / `engine:<name>` |

**响应：**

```json
{
  "project_id": 1,
  "framework_key": "pytorch",
  "baseline_run_id": "abc123",
  "baseline_run": {
    "run_id": "abc123",
    "name": "baseline-v1",
    "status": "completed",
    "architecture_id": 2,
    "engine": "ultralytics"
  }
}
```

### 7.2 设置对比基线

```
PUT /api/v2/projects/{project_id}/compare-baseline
```

**请求体：**

```json
{
  "framework_key": "pytorch",
  "baseline_run_id": "abc123"
}
```

### 7.3 清除对比基线

```
DELETE /api/v2/projects/{project_id}/compare-baseline?framework_key=pytorch
```
