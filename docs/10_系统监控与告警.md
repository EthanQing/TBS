# 系统监控与告警 API

> **路由前缀：**  
> - 系统监控：`/api/v2/system-metrics`（标签：`system-metrics`）  
> - 告警管理：`/api/v2/alarms`（标签：`alarms`）

---

## 一、系统监控 (`/api/v2/system-metrics`)

实时获取服务器 CPU、内存、GPU 等硬件资源使用情况。

### 1. 获取系统摘要

```
GET /api/v2/system-metrics/summary
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `node_id` | string | `backend` | 节点 ID，如 `backend` / `worker-yolo` |
| `node_type` | string | `backend` | 节点类型，如 `backend` / `worker` / `inference-worker` |

**响应示例：**

```json
{
  "node_id": "backend",
  "node_type": "backend",
  "timestamp": "2026-03-20T10:00:00+00:00",
  "cpu": {
    "usage_percent": 35.2,
    "core_count": 16,
    "frequency_mhz": 3600
  },
  "memory": {
    "total_gb": 64.0,
    "used_gb": 28.5,
    "usage_percent": 44.5
  },
  "disk": {
    "total_gb": 500.0,
    "used_gb": 210.0,
    "usage_percent": 42.0
  },
  "gpus": [
    {
      "gpu_id": 0,
      "name": "NVIDIA RTX 4090",
      "temperature_c": 65,
      "utilization_percent": 80,
      "memory_total_mb": 24576,
      "memory_used_mb": 18000,
      "memory_usage_percent": 73.2
    }
  ]
}
```

---

### 2. 获取历史数据

```
GET /api/v2/system-metrics/history
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `minutes` | int | 10 | 历史窗口（分钟），范围 1~1440 |
| `node` | string | `backend` | 节点 ID |
| `node_type` | string | `backend` | 节点类型 |
| `step_seconds` | int | 5 | 降采样步长（秒），范围 1~300 |

**响应示例：**

```json
{
  "node_id": "backend",
  "node_type": "backend",
  "minutes": 10,
  "step_seconds": 5,
  "points": [
    {
      "timestamp": "2026-03-20T09:50:00+00:00",
      "cpu_percent": 35.2,
      "memory_percent": 44.5,
      "gpu_utilization_percent": 80,
      "gpu_memory_percent": 73.2
    }
  ]
}
```

---

### 3. 获取集群概览

```
GET /api/v2/system-metrics/nodes
```

返回所有在线节点的状态概览。

**响应示例：**

```json
{
  "nodes": [
    {
      "node_id": "backend",
      "node_type": "backend",
      "status": "online",
      "last_seen": "2026-03-20T10:00:00+00:00"
    },
    {
      "node_id": "worker-yolo",
      "node_type": "worker",
      "status": "online",
      "last_seen": "2026-03-20T09:59:55+00:00"
    }
  ]
}
```

## 二、告警管理 (`/api/v2/alarms`)

告警模块基于规则引擎，监控训练任务指标并生成告警。

### 1. 告警规则类型

```
GET /api/v2/alarms/rule-types
```

获取系统支持的所有告警规则类型。

---

### 2. 规则管理

#### 2.1 获取规则列表

```
GET /api/v2/alarms/rules
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` / `page_size` | int | 分页参数 |
| `enabled` | bool | 按启用状态过滤 |

#### 2.2 创建规则

```
POST /api/v2/alarms/rules
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 规则名称 |
| `rule_type` | string | ✅ | 规则类型 |
| `severity` | string | ✅ | 告警级别 |
| `enabled` | bool | 否 | 是否启用（默认 true） |
| `config` | object | ✅ | 规则配置 |

**响应：** `201 Created`

#### 2.3 更新规则

```
PATCH /api/v2/alarms/rules/{rule_id}
```

#### 2.4 删除规则

```
DELETE /api/v2/alarms/rules/{rule_id}
```

---

### 3. 告警评估

```
POST /api/v2/alarms/evaluate
```

手动触发规则评估。

| 字段 | 类型 | 说明 |
|------|------|------|
| `run_ids` | string[] | 指定评估的训练任务列表（可选，为空则评估所有） |

**响应示例：**

```json
{
  "evaluated_rules": 5,
  "evaluated_runs": 3,
  "alerts_created": 1,
  "alerts_resolved": 0,
  "alerts": [
    {
      "alert_id": 10,
      "rule_id": 2,
      "severity": "warning",
      "source_id": "abc-123",
      "message": "训练任务 mAP 连续 5 个 epoch 未提升"
    }
  ]
}
```

---

### 4. 活跃告警

```
GET /api/v2/alarms/active
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `page` / `page_size` | int | 分页参数 |
| `severity` | string | 按严重度过滤 |
| `rule_type` | string | 按规则类型过滤 |
| `source_id` | string | 按来源 ID 过滤（如训练任务 ID） |

---

### 5. 确认告警

```
POST /api/v2/alarms/active/{alert_id}/ack
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `acked_by` | string | 确认者 |

---

### 6. 告警历史

```
GET /api/v2/alarms/history
```

参数同活跃告警列表。返回已解决的告警记录。

---

### 7. 告警汇总

```
GET /api/v2/alarms/summary
```

返回整体告警统计。

**响应示例：**

```json
{
  "total_active": 3,
  "total_resolved": 15,
  "by_severity": {
    "critical": 0,
    "warning": 2,
    "info": 1
  }
}
```

## 数据模型

### AlarmRuleOut

| 字段 | 类型 | 说明 |
|------|------|------|
| `rule_id` | int | 规则 ID |
| `name` | string | 规则名称 |
| `rule_type` | string | 规则类型 |
| `severity` | string | 告警级别 |
| `enabled` | bool | 是否启用 |
| `config` | object | 规则配置 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

### AlarmAlertOut

| 字段 | 类型 | 说明 |
|------|------|------|
| `alert_id` | int | 告警 ID |
| `rule_id` | int | 触发的规则 ID |
| `severity` | string | 告警级别 |
| `status` | string | 状态：`active` / `resolved` |
| `source_id` | string | 来源 ID（如训练任务 ID） |
| `message` | string | 告警消息 |
| `acked_by` | string? | 确认者 |
| `acked_at` | datetime? | 确认时间 |
| `resolved_at` | datetime? | 解决时间 |
| `created_at` | datetime | 创建时间 |
