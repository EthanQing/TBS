# 数据集管理 API

> **统一前缀：** `/api/v3`
> **标准数据集：** `/api/v3/standard-datasets`
> **违规数据集：** `/api/v3/illegal-datasets`
> **上传任务：** `/api/v3/dataset-upload-tasks`
> **导入目录：** `/api/v3/dataset-imports`

当前平台有两类数据集：

- **标准数据集**：训练可直接使用的数据集，通常是 YOLO 结构，供项目、训练任务、模型版本引用。
- **违规数据集**：外部标注平台或原始标注数据接入区，保留原始标签、版本、标签映射，后续可发布为标准数据集。

外部标注平台如果想直接创建数据集并上传数据，优先使用本文件第 1 节的对接流程。

---

## 1. 外部标注平台推荐对接流程

### 1.1 选择数据集类型

| 目标 | 推荐接口 | 说明 |
|------|----------|------|
| 上传后直接训练 | `POST /api/v3/standard-datasets` | 标准数据集只支持 `mode=upload` 覆盖导入 |
| 上传原始标注、后续清洗/映射/发布 | `POST /api/v3/illegal-datasets` | 违规数据集支持 `mode=upload` 和 `mode=append` |

### 1.2 创建数据集

创建标准数据集：

```http
POST /api/v3/standard-datasets
Content-Type: application/json
```

```json
{
  "name": "标注平台-车辆检测-20260610",
  "dataset_type": "detection",
  "format": "yolo",
  "description": "由外部标注平台同步"
}
```

创建违规数据集：

```http
POST /api/v3/illegal-datasets
Content-Type: application/json
```

```json
{
  "name": "标注平台-原始违规数据-20260610",
  "dataset_type": "detection",
  "format": "labelme",
  "description": "保留原始 LabelMe 标注，后续发布为标准数据集"
}
```

响应中会返回 `standard_dataset_id` 或 `illegal_dataset_id`，后续上传接口使用该 ID。

### 1.3 创建分片上传会话

标准数据集：

```http
POST /api/v3/standard-datasets/{standard_dataset_id}/upload-sessions
Content-Type: application/json
```

违规数据集：

```http
POST /api/v3/illegal-datasets/{illegal_dataset_id}/upload-sessions
Content-Type: application/json
```

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `filename` | string | 是 | ZIP 文件名，服务端只保留文件名部分 |
| `total_size` | int | 是 | ZIP 总字节数，必须大于 0 |
| `chunk_size` | int | 否 | 分片字节数；不传时使用服务端 `UPLOAD_CHUNK_SIZE_MB` |
| `mode` | string | 否 | `upload` 覆盖导入；违规数据集额外支持 `append` 追加版本 |
| `created_by` | string | 否 | 调用方或操作人 |
| `message` | string | 否 | 当前 schema 接收该字段；分片完成时违规数据集可在 complete 表单中再次传版本说明 |

示例：

```json
{
  "filename": "vehicle-detection.zip",
  "total_size": 1073741824,
  "chunk_size": 10485760,
  "mode": "upload",
  "created_by": "annotation-platform"
}
```

响应示例：

```json
{
  "session_id": "9f69f0aa-8c45-4cf3-9408-3b72d1ccaa3a",
  "dataset_kind": "standard",
  "dataset_id": 10001,
  "mode": "upload",
  "filename": "vehicle-detection.zip",
  "total_size": 1073741824,
  "chunk_size": 10485760,
  "total_parts": 103,
  "uploaded_parts": [],
  "status": "uploading",
  "created_by": "annotation-platform",
  "created_at": "2026-06-10T10:00:00+08:00",
  "updated_at": "2026-06-10T10:00:00+08:00",
  "expires_at": "2026-06-10T22:00:00+08:00",
  "error_message": null
}
```

### 1.4 上传分片

标准数据集：

```http
PUT /api/v3/standard-datasets/{standard_dataset_id}/upload-sessions/{session_id}/parts/{part_no}
Content-Type: multipart/form-data
```

违规数据集：

```http
PUT /api/v3/illegal-datasets/{illegal_dataset_id}/upload-sessions/{session_id}/parts/{part_no}
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | 是 | 当前分片内容 |

规则：

- `part_no` 从 1 开始，到 `total_parts` 结束。
- 除最后一片外，每片大小必须等于 `chunk_size`。
- 最后一片大小必须等于 `total_size - chunk_size * (total_parts - 1)`。
- 分片可重传，同一 `part_no` 会覆盖对应临时分片。

响应示例：

```json
{
  "session_id": "9f69f0aa-8c45-4cf3-9408-3b72d1ccaa3a",
  "part_no": 1,
  "size": 10485760,
  "uploaded_parts": [1],
  "status": "uploading"
}
```

### 1.5 查询或取消上传会话

查询标准数据集上传会话：

```http
GET /api/v3/standard-datasets/{standard_dataset_id}/upload-sessions/{session_id}
```

查询违规数据集上传会话：

```http
GET /api/v3/illegal-datasets/{illegal_dataset_id}/upload-sessions/{session_id}
```

取消标准数据集上传会话：

```http
DELETE /api/v3/standard-datasets/{standard_dataset_id}/upload-sessions/{session_id}
```

取消违规数据集上传会话：

```http
DELETE /api/v3/illegal-datasets/{illegal_dataset_id}/upload-sessions/{session_id}
```

### 1.6 完成上传并启动后台导入

标准数据集：

```http
POST /api/v3/standard-datasets/{standard_dataset_id}/upload-sessions/{session_id}/complete
```

违规数据集：

```http
POST /api/v3/illegal-datasets/{illegal_dataset_id}/upload-sessions/{session_id}/complete
Content-Type: multipart/form-data
```

违规数据集 complete 可选表单字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | string | 否 | 版本说明 |

响应状态码为 `202 Accepted`，表示已经创建后台任务：

```json
{
  "task_id": "f487d9d4-bd45-48f6-8f75-5fb1b45f8f6f",
  "session_id": "9f69f0aa-8c45-4cf3-9408-3b72d1ccaa3a",
  "status": "queued"
}
```

注意：`complete` 返回时数据集还不一定可用，必须继续查询上传任务。

### 1.7 轮询后台上传任务

```http
GET /api/v3/dataset-upload-tasks/{task_id}
```

响应示例：

```json
{
  "task_id": "f487d9d4-bd45-48f6-8f75-5fb1b45f8f6f",
  "dataset_kind": "illegal",
  "dataset_id": 20001,
  "session_id": "9f69f0aa-8c45-4cf3-9408-3b72d1ccaa3a",
  "mode": "append",
  "source_type": "zip",
  "status": "indexing",
  "stage": "indexing",
  "progress": 88,
  "error_message": null,
  "created_by": "annotation-platform",
  "message": "追加 2026-06-10 批次",
  "created_at": "2026-06-10T10:03:00+08:00",
  "updated_at": "2026-06-10T10:05:00+08:00",
  "finished_at": null
}
```

常见状态/阶段：

| 值 | 说明 |
|----|------|
| `queued` | 任务已创建，等待执行 |
| `extracting` | ZIP 解压中 |
| `validating` | 校验数据结构和内容 |
| `materializing` | 违规数据集物化版本内容 |
| `indexing` | 建立图片、标签、统计或 manifest 索引 |
| `finalizing` | 收尾并激活版本 |
| `done` | 处理完成 |
| `failed` | 处理失败，查看 `error_message` |

## 2. 数据包格式建议

### 2.1 标准数据集 ZIP

标准数据集建议上传 YOLO 结构：

```text
dataset.zip
  data.yaml
  images/
    train/
    val/
  labels/
    train/
    val/
```

### 2.2 违规数据集 ZIP

违规数据集通常用于保存外部标注平台原始数据。LabelMe/JSON 类数据建议保持图片与 JSON 同名：

```text
dataset.zip
  images/
    000001.jpg
    000002.jpg
  json/
    000001.json
    000002.json
```

后续发布为标准数据集时，服务端会按图片/JSON 基名配对。缺图片、缺 JSON、图片截断或不可解码的样本会记录为跳过项和 warning；整批没有任何可用配对样本或没有生成有效 YOLO 样本时发布会失败。

| 字段 | 类型 | 必填 | 说明 |

### 3.4 标准数据集分片上传

见第 1 节。标准数据集只支持 `mode=upload`；传 `append` 会返回校验错误。

### 3.5 从服务端目录导入标准数据集

```http
POST /api/v3/standard-datasets/{standard_dataset_id}/import-from-path
Content-Type: application/json
```

请求体：

| 字段 | 类型 | 必填 | 默认值 | 说明 |

响应同 `DatasetUploadCompleteOut`，返回 `task_id` 后继续查询 `/api/v3/dataset-upload-tasks/{task_id}`。

### 3.7 标准数据集拆分

执行拆分：

```http
POST /api/v3/standard-datasets/{standard_dataset_id}/split
Content-Type: application/json
```

请求体：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `train_ratio` | float | 否 | 0.9 | 训练集比例，0~1 |
| `val_ratio` | float | 否 | - | 验证集比例 |
| `test_ratio` | float | 否 | - | 测试集比例 |
| `seed` | int | 否 | - | 随机种子 |
| `shuffle` | bool | 否 | true | 是否打乱 |
| `overwrite` | bool | 否 | true | 是否覆盖已有拆分 |

查询拆分结果：

```http
GET /api/v3/standard-datasets/{standard_dataset_id}/split?page=1&page_size=50&split=train
```

### 3.8 标准数据集事件

```http
GET /api/v3/standard-datasets/{standard_dataset_id}/events?page=1&page_size=50
```

## 4. 违规数据集接口

### 4.1 创建违规数据集


| `format` | string | 否 | 数据格式，默认 `yolo`，也可记录 `labelme` 等原始来源格式 |

### 4.2 获取违规数据集列表

```http
GET /api/v3/illegal-datasets
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `page` | int | 1 | 页码 |
| `page_size` | int | 50 | 每页条数，1~500 |
| `format` | string | - | 按格式过滤 |

### 4.3 获取、更新、删除违规数据集

```http
GET /api/v3/illegal-datasets/{illegal_dataset_id}
GET /api/v3/illegal-datasets/{illegal_dataset_id}/detail?versions_limit=20&events_limit=20
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| 字段 | 类型 | 必填 | 默认值 | 说明 |

### 4.8 违规数据集统计与轻量兼容

```http
GET /api/v3/illegal-datasets/{illegal_dataset_id}/statistics?version_id=30001
```

违规数据集已取消文件列表、版本文件打开、图片标注查看、样本预览和缩略图生成。详情页不再调用 `/files`、`/versions/{version_id}/files/{file_path}`、`/image-annotations`、`/view` 或 `/api/v3/thumbnails/illegal/...`。`/view` 仅为兼容旧调用保留，返回空 `categories` 和 `items`。

### 4.9 违规标签与标签映射

获取原始标签：

```http
GET /api/v3/illegal-datasets/{illegal_dataset_id}/raw-labels
```

获取当前映射：

```http
GET /api/v3/illegal-datasets/{illegal_dataset_id}/label-mappings
```

保存映射：

```http
PUT /api/v3/illegal-datasets/{illegal_dataset_id}/label-mappings
Content-Type: application/json
```

请求体：

```json
{
  "items": [
    {
      "raw_label": "车辆%小汽车",
      "mapped_label": "car",
      "status": "keep"
    },
    {
      "raw_label": "忽略区域",
      "mapped_label": "",
      "status": "delete"
    }
  ]
}
```

说明：

- `status=keep` 时 `mapped_label` 必填。
- `status=delete` 表示发布标准数据集时丢弃该标签。
- 后端会对原始标签做规范化去重，避免大小写、空白或特殊字符导致重复映射冲突。

### 4.10 发布违规数据集为标准数据集

```http
POST /api/v3/illegal-datasets/{illegal_dataset_id}/publish-jobs
Content-Type: application/json
```

请求体：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 发布后的标准数据集名称 |
| `description` | string | 否 | 发布后的标准数据集描述 |

## 7. 对接注意事项

- 所有当前接口路径均为 `/api/v3`，不要再调用历史 `/api/v2/datasets`。
- 标准数据集和违规数据集的 ID 字段不同：分别是 `standard_dataset_id` 和 `illegal_dataset_id`。
- 分片上传完成后必须轮询 `/api/v3/dataset-upload-tasks/{task_id}`，直到 `status=done` 才认为数据集可用。
- 标准数据集不支持追加上传；需要追加原始数据时使用违规数据集 `mode=append`。
- 违规数据集版本、统计和发布依赖 `manifest_path`。违规数据集已取消文件列表、原图打开、图片标注查看、样本预览和缩略图生成；外部导入后如果任务失败，应优先查看上传任务的 `error_message`。
- 列表接口按 `updated_at DESC` 分页；大数据量调用时应控制 `page_size`，标准数据集引用场景使用 `include_statistics=false`。
