# 辅助接口 API

本文档涵盖平台的辅助性 API 接口，包括网络架构、框架插件、预训练模型上传、缩略图、图表配置及平台统计。

---

## 一、网络架构 (`/api/v2/architectures`)

> **标签：** `architectures`

### 1. 获取架构列表

```
GET /api/v2/architectures
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `family` | string | 否 | 按模型族过滤（如 `yolo`） |
| `task_type` | string | 否 | 按任务类型过滤：`detection` / `segmentation` / `classification` |
| `q` | string | 否 | 搜索关键词 |

**响应示例：**

```json
[
  {
    "architecture_id": 1,
    "name": "YOLOv8n",
    "family": "yolo",
    "variant": "yolov8n",
    "task_type": "detection",
    "engine": "ultralytics",
    "description": "YOLOv8 Nano 目标检测模型",
    "created_at": "2026-03-01T00:00:00+00:00"
  }
]
```

---

### 2. 创建架构

```
POST /api/v2/architectures
```

**响应：** `201 Created`

---

## 二、框架插件 (`/api/v2/frameworks`)

> **标签：** `frameworks`

### 1. 获取框架列表

```
GET /api/v2/frameworks
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `implemented` | bool | 按实现状态过滤（不传返回全部） |

**响应示例：**

```json
[
  {
    "plugin_id": "ultralytics_yolo",
    "name": "Ultralytics YOLO",
    "engine": "ultralytics",
    "description": "Ultralytics YOLOv8/v11 训练引擎",
    "implemented": true,
    "supported_tasks": ["detection", "segmentation", "classification"]
  },
  {
    "plugin_id": "paddle_detection",
    "name": "PaddleDetection",
    "engine": "paddle",
    "description": "PaddlePaddle 检测框架",
    "implemented": true,
    "supported_tasks": ["detection"]
  }
]
```

---

### 2. 获取框架配置 Schema

```
GET /api/v2/frameworks/{plugin_id}/config-schema
```

返回指定框架插件的配置项 JSON Schema，用于前端动态生成配置表单。

**响应示例：**

```json
{
  "plugin_id": "ultralytics_yolo",
  "schema": {
    "type": "object",
    "properties": {
      "epochs": { "type": "integer", "default": 100 },
      "batch_size": { "type": "integer", "default": 16 },
      "image_size": { "type": "integer", "default": 640 }
    }
  }
}
```

---

### 3. 校验框架配置

```
POST /api/v2/frameworks/{plugin_id}/validate-config
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `config` | object | ✅ | 待校验的配置对象 |

**响应：**

```json
{
  "valid": true,
  "normalized_config": { "epochs": 100, "batch_size": 16 },
  "warnings": []
}
```

## 三、预训练模型上传 (`/api/v2/pretrain-models`)

> **标签：** `pretrain-models`

### 上传预训练权重

```
POST /api/v2/pretrain-models/upload
```

**Content-Type:** `multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | ✅ | 权重文件（支持 `.pt` / `.pth` / `.ckpt`） |

**响应：** `201 Created`

```json
{
  "token": "pretrain_uploads/model-abc123.pt",
  "path": "/static/temp/pretrain_uploads/model-abc123.pt",
  "filename": "model.pt"
}
```

> 返回的 `token` 可在创建训练任务时作为预训练权重引用。

## 四、缩略图 (`/api/v2/thumbnails`)

> **标签：** `thumbnails`

### 获取缩略图

```
GET /api/v2/thumbnails/{dataset_id}/{file_path}
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `dataset_id` | int | 数据集 ID |
| `file_path` | string | 图片相对路径（如 `images/train/001.jpg`） |
| `size` | int | 缩略图最大边长（默认 200，范围 16~1024） |
| `version_id` | int | 可选，数据集版本 ID |

**响应：** 图片二进制流（`image/jpeg` 或对应 MIME 类型）

**调用示例：**

```
GET /api/v2/thumbnails/1/images/train/img_001.jpg?size=200
```

---

## 五、图表配置 (`/api/v2/chart-configs`)

> **标签：** `chart-configs`

用于持久化前端图表的用户自定义配置（如选中指标、坐标轴范围等）。

### 1. 获取配置

```
GET /api/v2/chart-configs/{scope}
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `scope` | string | 配置作用域（如 `training-metrics`、`compare-view`） |

**响应：** 配置 JSON 对象（未保存过则返回 `{}`）

---

### 2. 保存配置

```
PUT /api/v2/chart-configs/{scope}
```

**请求体：** 任意 JSON 对象

**响应：**

```json
{
  "ok": true
}
```

## 六、平台统计 (`/api/v2/stats`)

> **标签：** `stats`

### 获取平台汇总统计

```
GET /api/v2/stats/summary
```

**响应示例：**

```json
{
  "datasets": 15,
  "projects": 8,
  "training_runs_total": 120,
  "training_runs_by_status": {
    "created": 2,
    "queued": 1,
    "running": 3,
    "completed": 95,
    "failed": 15,
    "cancelled": 4
  },
  "model_versions": 45,
  "deployments": 10,
  "deployments_active": 5
}
```

### StatsSummary 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `datasets` | int | 数据集总数 |
| `projects` | int | 项目总数 |
| `training_runs_total` | int | 训练任务总数 |
| `training_runs_by_status` | object | 按状态分组的训练任务数量 |
| `model_versions` | int | 模型版本总数 |
| `deployments` | int | 部署总数 |
| `deployments_active` | int | 活跃部署数量 |
