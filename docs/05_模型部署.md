# 模型部署 API (Deployments)

> **路由前缀：** `/api/v2/deployments` 和 `/api/v2/deployment-runs`  
> **标签：** `deployments`、`deployment-runs`

部署模块负责将注册的模型版本部署到目标环境，支持执行、回滚、状态跟踪和日志管理。

---

## 一、部署管理 (`/api/v2/deployments`)

### 1. 获取部署列表

```
GET /api/v2/deployments
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码 |
| `page_size` | int | 否 | 50 | 每页条数 |
| `project_id` | int | 否 | - | 按项目过滤 |
| `model_version_id` | int | 否 | - | 按模型版本过滤 |
| `status` | string | 否 | - | `pending`/`deploying`/`active`/`inactive`/`failed`/`deleting` |
| `is_active` | bool | 否 | - | 是否已激活 |

---

### 2. 创建部署

```
POST /api/v2/deployments
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model_version_id` | int | ✅ | 模型版本 ID |
| `name` | string | ✅ | 部署名称 (1~255) |
| `platform` | string | ✅ | 平台：`local`/`docker`/`kubernetes`/`aws`/`azure`/`gcp` |
| `config` | object | 否 | 部署配置 |
| `health_check_url` | string | 否 | 健康检查 URL |

**请求示例：**

```json
{
  "model_version_id": 1,
  "name": "行人检测-生产部署",
  "platform": "local",
  "config": { "port": 8080 }
}
```

**响应：** `201 Created`，返回 `DeploymentOut` 对象。

---

### 3. 执行部署

```
POST /api/v2/deployments/{deployment_id}/execute
```

触发部署执行流程，创建部署运行记录，同时可生成 API Key。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `operator` | string | 否 | 操作人（默认 `admin`） |
| `reason` | string | 否 | 部署原因 |
| `rotate_api_key` | bool | 否 | 是否轮换 API Key（默认 true） |
| `conf` | float | 否 | 置信度阈值（默认 0.25） |
| `iou` | float | 否 | IOU 阈值（默认 0.45） |

**响应示例：**

```json
{
  "run": {
    "run_id": "deploy-run-001",
    "deployment_id": 1,
    "project_id": 1,
    "model_version_id": 1,
    "trigger_type": "manual",
    "status": "queued",
    "phase": "preparing",
    "progress": 0,
    "cancel_requested": false,
    "created_at": "2026-03-15T10:00:00+00:00",
    "updated_at": "2026-03-15T10:00:00+00:00"
  },
  "issued_api_key": "sk-xxxxxxxxxxxx",
  "api_key_hint": "sk-xxxx...xxxx"
}
```

> ⚠️ `issued_api_key` 仅在创建时返回一次，请妥善保存，用于后续调用在线推理 API。

---

### 4. 获取部署详情

```
GET /api/v2/deployments/{deployment_id}
```

---

### 5. 更新部署

```
PATCH /api/v2/deployments/{deployment_id}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 名称 |
| `status` | string | 状态 |
| `endpoint_url` | string | 端点 URL |
| `health_check_url` | string | 健康检查 URL |
| `config` | object | 配置 |
| `is_active` | bool | 是否激活 |

---

### 6. 删除部署

```
DELETE /api/v2/deployments/{deployment_id}
```

---

### 7. 部署日志

#### 获取日志

```
GET /api/v2/deployments/{deployment_id}/logs?limit=200
```

#### 添加日志

```
POST /api/v2/deployments/{deployment_id}/logs
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `level` | string | 日志级别：`DEBUG`/`INFO`/`WARNING`/`ERROR` |
| `message` | string | 日志内容 |
| `data` | object | 附加数据 |

---

### 8. 部署回滚

#### 8.1 获取回滚候选模型

```
GET /api/v2/deployments/{deployment_id}/rollback/candidates
```

返回当前部署可回滚到的历史模型版本列表。

#### 8.2 执行回滚

```
POST /api/v2/deployments/{deployment_id}/rollback
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target_model_version_id` | int | ✅ | 回滚目标模型版本 ID |
| `reason` | string | ✅ | 回滚原因 |
| `operator` | string | 否 | 操作人（默认 `admin`） |

**响应：**

```json
{
  "deployment": { "deployment_id": 1, "..." : "..." },
  "event": {
    "log_id": 5,
    "deployment_id": 1,
    "created_at": "2026-03-20T10:00:00+00:00",
    "operator": "admin",
    "reason": "模型精度下降",
    "from_model_version_id": 2,
    "to_model_version_id": 1,
    "from_version": "v2.0",
    "to_version": "v1.0"
  }
}
```

#### 8.3 回滚历史

```
GET /api/v2/deployments/{deployment_id}/rollback/history?limit=200
```

## 二、部署运行 (`/api/v2/deployment-runs`)

### 1. 获取部署运行详情

```
GET /api/v2/deployment-runs/{run_id}
```

---

### 2. 重试部署运行

```
POST /api/v2/deployment-runs/{run_id}/retry
```

---

### 3. 取消部署运行

```
POST /api/v2/deployment-runs/{run_id}/cancel
```

---

### 4. 获取运行日志

```
GET /api/v2/deployment-runs/{run_id}/logs?from_seq=0&limit=1000
```

---

### 5. 部署运行实时流 (WebSocket)

```
ws://<host>/api/v2/deployment-runs/{run_id}/stream?from_seq=0
```

#### 消息类型

| type | 说明 |
|------|------|
| `snapshot` | 初始运行快照 |
| `progress` | 进度/状态更新 |
| `log` | 逐条日志 |
| `done` | 运行结束 |
| `error` | 错误 |
| `ping` | 心跳 |

#### 部署运行阶段 (phase)

| 阶段 | 说明 |
|------|------|
| `preparing` | 准备中 |
| `validate_artifacts` | 验证产物 |
| `materialize_runtime` | 构建运行时 |
| `smoke_test` | 冒烟测试 |
| `activate` | 激活 |
| `done` | 完成 |
| `cancelled` | 已取消 |

---

## 数据模型

### DeploymentOut

| 字段 | 类型 | 说明 |
|------|------|------|
| `deployment_id` | int | 部署 ID |
| `model_version_id` | int | 模型版本 ID |
| `name` | string | 名称 |
| `platform` | string | 部署平台 |
| `status` | string | 状态 |
| `endpoint_url` | string? | 服务端点 |
| `health_check_url` | string? | 健康检查 URL |
| `config` | object? | 配置 |
| `api_key_hint` | string? | API Key 提示（脱敏） |
| `is_active` | bool | 是否激活 |
| `deployed_at` | datetime? | 部署时间 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

### DeploymentRunOut

| 字段 | 类型 | 说明 |
|------|------|------|
| `run_id` | string | 运行 ID |
| `deployment_id` | int | 部署 ID |
| `project_id` | int | 项目 ID |
| `model_version_id` | int | 模型版本 ID |
| `trigger_type` | string | 触发类型：`manual` |
| `status` | string | 状态：`queued`/`running`/`completed`/`failed`/`cancelled` |
| `phase` | string | 执行阶段 |
| `current_step` | string? | 当前步骤 |
| `progress` | int | 进度百分比 |
| `cancel_requested` | bool | 是否已请求取消 |
| `error_message` | string? | 错误信息 |
| `snapshot` | object? | 运行快照 |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |
